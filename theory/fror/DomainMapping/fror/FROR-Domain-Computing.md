# FROR в вычислительных системах

## Отображение канона

- `Ω`: глобальные микросостояния ИС (БД, кэши, очереди, внешние эффекты).
- `Σ`: фиксируемые представления (заказы, статусы, агрегаты, логи).
- `F`: материализация/проекция событий в наблюдаемые состояния.
- `c`: стоимость изменения фиксации (запись, синхронизация, компенсация).
- `τ`: накопленная операционная стоимость траектории.
- `G`: допустимые переходы бизнес- и системной логики.
- `G⁰`: переходы без роста `τ` в выбранной нормализации.
- `SLOT`: локально устойчивые режимы сервиса.

## Наблюдаемые операторы

- `O_c`: latency + write-amplification + стоимость координации.
- `O_tau`: сумма `c` по цепочке изменений.
- `O_G0`: доля переходов без новых затрат.
- `O_SLOT`: conductance + survival на графе состояния.

## Интерпретация rollback

`rollback` обычно не равен полному возврату глобального состояния.
Практически используется компенсация: новый согласованный переход,
синхронизированный с текущим контекстом.
