## 0. Статус и область

- Уровень A: структурная формулировка порога.
- Уровень B: эмпирическая проверка в конкретной нормализации.

Документ обобщает memory-only профиль: память `M_O` трактуется как
бит-нормализация общего ресурса различения `R_O`.

------------------------------------------------------------------------

## 1. Базовые определения

- Наблюдатель: `O = (R_O, Sigma_O, pi_O)`.
- `R_O`: эффективный ресурс различения (доменная нормализация).
- `Sigma_O`: множество различимых внутренних состояний.
- `pi_O`: стратегия coarse-graining в классы `P = {P_1, ..., P_K}`.
- Симметрия классов: инвариантность распределения классов к перестановкам в `D`.
- `DeltaS_defect`: изменение дефектной энтропии на актах фиксации.

Информационная нормализация:
- `R_O := |Sigma_O|` (каноническая дискретная запись),
- `M_O := ceil(log2 R_O)` (бит-эквивалент),
- при общей нормализации используется `K(R_O)` как число одновременно
  различимых альтернатив.

Практическое чтение метрик:
- `R_O` отвечает на вопрос "сколько внутренних различимых состояний есть";
- `M_O` даёт memory-oriented прокси той же величины;
- `K(R_O)` отвечает на вопрос "сколько альтернатив реально удерживается
  раздельно в выбранной нормализации".

Поэтому threshold-файл не должен читаться как memory-only теорема:
память здесь является частной и удобной нормализацией общего ресурса
различения.

Под "допустимой стратегией" понимается функция, вычислимая конечным
детерминированным автоматом без внешнего оракула, внешней памяти и внешней
случайности.

------------------------------------------------------------------------

## 2. Режим трихотомии и его спецификация

Используется трёхклассовая структура:

`D = {-, 0, +}`.

Режим должен быть явно указан (C5):
- `A3-mode`: трихотомия задана аксиоматически;
- `Derived-mode`: трихотомия является производной конструкцией.

Бинарный режим `0/1` допускается как деградационный режим детекции, но не
считается достаточным для нетривиальной направленной динамики сложности.

Для операционной согласованности необходимо разделять:
- `0_structural`: структурная неприменимость/неразличимость в данном контексте;
- `0_no_cost_transition`: переход без роста `tau` при `c_fix = 0`.

Эти два класса не должны смешиваться в протоколе измерений.

------------------------------------------------------------------------

## 3. Лемма о нижней границе ресурса

Для устойчивого различения всех классов `{- ,0,+}` необходимо:

`K(R_O) >= 3`.

В канонической дискретной записи (`K(R)=R`):

`R_O >= 3`.

В бит-нормализации:

`M_O >= ceil(log2 3) = 2`.

Это комбинаторный порог репрезентации (необходимое условие).

------------------------------------------------------------------------

## 4. Теорема о пороге ресурса и росте дефектной энтропии

### Теорема

Пусть:
1. используется трихотомный режим `{- ,0,+}` с явно зафиксированным
   `mode_trichotomy` (`A3-mode` или `Derived-mode`);
2. coarse-graining неинъективен на релевантных альтернативах;
3. акты фиксации выбора имеют положительную цену (`c_fix > 0` на `C_fix`);
4. отсутствуют внешний оракул, внешняя память и внешняя случайность.

Тогда:
- при `K(R_O) >= 3` (эквивалентно `R_O >= 3`, или `M_O >= 2` в бит-нормализации)
  существует допустимая стратегия, нарушающая симметрию классов;
- на актах платной фиксации наблюдается `DeltaS_defect > 0`;
- при `K(R_O) < 3` устойчивое трихотомическое различение недостижимо, и
  нетривиальное нарушение симметрии в этом режиме не реализуется.

### Важная оговорка (необходимость vs достаточность)

Порог `K(R_O) >= 3` является необходимым, но сам по себе не достаточным.
Для наблюдаемой асимметрии требуются дополнительные динамические
предпосылки (правило выбора/фиксации, неинвариантное относительно
перестановок классов).

------------------------------------------------------------------------

## 5. Связь с tau и ценой фиксации

Рост `tau` определяется только ценой фиксации:

`tau_{t+1} = tau_t + c_fix(sigma_t, sigma_{t+1})`.

Следовательно:
- `DeltaS_defect > 0` ожидается на платных актах фиксации;
- шаги `0_no_cost_transition` не обязаны увеличивать `tau`;
- фоновая диссипация `c_bg > 0` учитывается отдельно и не подменяет
  вклад `c_fix` в `tau`.

------------------------------------------------------------------------

## 6. Экспериментальный профиль (минимум)

### 6.1 Цель

Проверить одновременно:
1. порог `K(R_O) >= 3` для трихотомного режима;
2. рост `DeltaS_defect` именно на платных актах фиксации.

### 6.2 Обязательные поля протокола

- `mode_trichotomy` (`A3-mode` или `Derived-mode`);
- явное правило маппинга событий в `{- ,0,+}`;
- явное разделение `0_structural` и `0_no_cost_transition`;
- `R_O` и/или `K(R_O)` (при необходимости также `M_O`);
- признаки `paid_fixation`, `tau_before`, `tau_after`,
  `DeltaS_before`, `DeltaS_after`.

### 6.3 Критерий фальсификации

Связка "порог ресурса + рост дефектной энтропии" опровергается для данной
нормализации, если наблюдается хотя бы одно:
1. устойчивое трихотомическое различение при `K(R_O) < 3` без внешнего оракула;
2. отсутствие роста `DeltaS_defect` на платных фиксациях при `K(R_O) >= 3`;
3. систематический рост `DeltaS_defect` в контрольных режимах, где платной
   фиксации/неинъективного склеивания нет.

------------------------------------------------------------------------

## 7. Короткая формулировка

Память является частной нормализацией общего ресурса различения.
Минимальный порог нетривиального трихотомического режима задается как
`K(R_O) >= 3` (или `M_O >= 2` в бит-нормализации). Рост дефектной энтропии
возникает из неинъективного coarse-graining с платной фиксацией, а не из
самого факта классификации.
