# PhaseTransition-LC

## 1. Определение

LC-PhaseTransition — автономный процесс перехода между фазами LifeCycle,
характеризующийся утратой устойчивости предыдущего режима
и формированием нового доминирования.

Переход считается завершённым,
когда прежний режим становится статистическим исключением.

## 2. Свойства

- внутреннее время τ
- состояние In-Transition
- параметр порядка (Order Parameter, OP)
- результат result ∈ {true, false}
- возможен absorbing state

## 2.1 Ответственность в переходе

В `LC-PT` до завершения перехода одновременно участвуют минимум:

1. `from_phase_owner` (предыдущий режим);
2. `to_phase_owner` (формирующийся режим).

Нормативно:

1. старт новой LC-фазы не является достаточным условием закрытия `LC-PT`;
2. `LC-PT` считается завершенным только при подтверждении устойчивости
   нового режима по параметрам перехода (`OP`, τ, доменные критерии).

## 3. Классификация по условности

### Безусловный переход

Переход инициируется при завершении предыдущей фазы
без необходимости проверки дополнительных условий
или запуска дополнительных подпроцессов.

result = true по определению,
если предыдущая фаза завершена успешно.

### Условный переход

Переход требует выполнения дополнительного условия
или процесса (метрика, ресурс, взаимодействие).

## 4. Метрики (канон)

- τ — внутреннее время
- Σ — стоимость
- OP — параметр порядка
- result — булев результат

Доменные определения задаются через [[DomainLexicon]].
