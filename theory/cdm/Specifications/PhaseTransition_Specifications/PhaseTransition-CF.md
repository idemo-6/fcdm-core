# PhaseTransition-CF

## 1. Определение

CF-PhaseTransition — автономный процесс передачи,
согласования и подготовки результатов между фазами ChangeFlow,
обеспечивающий возможность запуска следующей фазы.

Переход не обязательно связан с изменением состояния системы.

## 2. Свойства

- может иметь внутреннее время τ
- может иметь состояние In-Transition
- результат result ∈ {true, false}
- отказ приводит к прерыванию текущего ChangeFlow

## 3. Поведение при отказе

Если result = false:

- текущий ChangeFlow считается прерванным
- инициируется фаза Evaluate (CF6)

## 4. Evaluate как формирование опыта

Evaluate — фаза формирования опыта системы:

- фиксирует результат взаимодействия
- обновляет внутренние модели
- завершает ChangeFlow независимо от успеха

## 5. Классификация по условности

### Безусловный переход

Переход инициируется при завершении предыдущей фазы
без дополнительных условий или подпроцессов.

Пример: CF5 ⇒ CF6 при успешном завершении implement.

### Условный переход

Переход требует дополнительного процесса
(транспорт, согласование, проверка, взаимодействие).

## 6. Метрики (канон)

- τ — время перехода
- Σ — стоимость
- result — булев результат

Доменные расширения задаются через [[DomainLexicon]].
