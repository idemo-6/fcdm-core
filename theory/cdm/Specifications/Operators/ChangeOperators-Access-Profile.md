# Профиль: ChangeOperators Access & Phase Mapping

## 1. Назначение

Профиль задает прикладную классификацию операторов:
- по источнику доступа;
- по фазовой применимости;
- по базовым safety-ограничениям.

---

## 2. Нормативная граница

Это профиль, а не канон.

Опорные документы:
- `ChangeOperators (каноническая)`
- `Intent (каноническая)`
- `Context (каноническая)`
- `ChangeFlow-6 (каноническая)`

---

## 3. Классификация по доступу (L/P/S/E)

| Тип | Обозначение | Источник |
|---|---|---|
| Local | `L` | локальный контур системы |
| Parent | `P` | родительский контур/наследование политики |
| Sibling | `S` | смежные подсистемы через согласованный интерфейс |
| External | `E` | внешние контуры/системы |

Замечание:
- это операционный профиль доступа, не онтологическая типология систем.

---

## 4. Базовое фазовое соответствие (минимум)

| Фаза CF | Профильный минимум класса операторов |
|---|---|
| `CF1` (`collect`) | наблюдательные/сигнальные |
| `CF2` (`analyze`) | интерпретационные/логико-модельные |
| `CF3` (`forecast`) | прогностические/симуляционные |
| `CF4` (`decide`) | селекционные/разрешение неопределенности |
| `CF5` (`implement`) | исполнительные/фиксация изменения |
| `CF6` (`evaluate`) | оценочные/метрики и запись опыта |

Это карта минимальной достаточности; конкретные операторы определяются доменным `OperatorSet`.

---

## 5. Safety-нормы профиля

1. Оператор с меткой `forbidden` не может входить в `pi_ops`.
2. `CF5` не допускает операторы, нарушающие заданные policy-ограничения authority.
3. В `CF6` обязателен trace использованных операторов и причины блокировок.

---

## 6. Заключение

Профиль связывает канонический слой операторов с прикладной схемой доступа и фазовой дисциплиной без изменения семантической первичности Intent.
