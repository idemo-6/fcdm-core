# Спецификация: CDM v2 — Теорема минимальности CF-6 и связь с LC-6

> Документ формализует тезис о минимальной функциональной декомпозиции CF-6
> и задает связь с LC-6 как макро-динамикой повторяющихся циклов.
> Краткая цитируемая версия: [[fcdm-core/theory/cdm/Specifications/CDM-v2-Minimality-Canonical-Short|CDM-v2-Minimality-Canonical-Short]].

---

## 1. Исходные определения

### 1.1 Адаптивный вычислитель

Пусть система:

`S = (Omega, R, M, Pi)`,

где:

- `Omega` — пространство доступных альтернатив;
- `R` — ограниченный ресурс разрешения;
- `M` — внутренняя модель мира;
- `Pi` — стратегия поведения.

Система считается адаптивной, если она:

- получает данные о состоянии среды;
- изменяет внутреннюю модель;
- выбирает действие;
- изменяет внешнее состояние;
- корректирует поведение по результату.

### 1.2 Необратимое разрешение неопределенности

Пусть до шага управления:

`T_k = {t1, t2, ..., tn}`, `n > 1`.

После шага:

`T_(k+1) subset T_k`.

Если `|T_(k+1)| < |T_k|`, произошла редукция альтернатив.
Если возврат отброшенных альтернатив требует дополнительных затрат, редукция
считается необратимой.

Операторная форма редукции шага:

`Phi_k: T_k -> T_(k+1)`, где `|T_(k+1)| < |T_k|`.

Профильно (опционально) может вводиться лог-метрика редукции:

`tau_red(k) = log(|T_k| / |T_(k+1)|)`.

Важно:
`tau_red` является производной метрикой конкретной нормализации и
не переопределяет каноническую метрику стоимости `tau` в спецификациях CF/LC.

### 1.3 Целевая функция

Минимальная цель адаптивного вычислителя:

`min_(Pi) E[tau_future]`,

где `tau` — накопленная необратимая стоимость.

---

## 2. Шесть необходимых функций адаптивного шага

Вводятся шесть функционально различных операторов:

- `C` (collect): `C: W -> D`;
- `A` (analyze): `A: D -> S_hat`;
- `F` (forecast): `F: S_hat -> {T_i}`;
- `Dc` (decide): `Dc: {T_i} -> T*`;
- `I` (implement): `I: T* -> W'`;
- `E` (evaluate): `E: (W', M) -> DeltaM`.

Интерпретация:

- `C` связывает систему с текущим состоянием мира;
- `A` превращает данные в структурированное состояние;
- `F` порождает пространство траекторий;
- `Dc` выполняет селекцию (основной tau-акт);
- `I` материализует выбор в мире;
- `E` обновляет модель по результату.

---

## 3. Несливаемость функций (условие общности)

Для универсальной модели адаптивности требуются различения:

- `collect != analyze` (сигнал и его интерпретация);
- `analyze != forecast` (текущее состояние и будущие траектории);
- `forecast != decide` (множество вариантов и селекция);
- `decide != implement` (выбор и физическое действие);
- `implement != evaluate` (действие и оценка последствий);
- `evaluate != collect` (мета-оценка цикла и новый сырой сенсорный ввод).

Доменные свертки допустимы только как представление, но не как устранение
функционального класса.

---

## 4. Теорема минимальности CF-6

### Формулировка

Для универсальной адаптивной системы, выполняющей необратимое разрешение
неопределенности при ограниченном ресурсе, минимальный полный функциональный
цикл содержит не менее шести классов операций:

`{C, A, F, Dc, I, E}`.

То есть каноническая форма:

`CF-6 = (collect, analyze, forecast, decide, implement, evaluate)`.

### Доказательство (по необходимости)

Удаление любого элемента из `{C, A, F, Dc, I, E}` убирает обязательный класс
адаптивности:

- без `C` нет связи со средой;
- без `A` данные остаются шумом;
- без `F` нет пространства альтернатив;
- без `Dc` нет редукции;
- без `I` нет воздействия на мир;
- без `E` нет обучения и перенастройки модели.

Следовательно, цикл с числом функций `< 6` либо не является универсально
адаптивным, либо содержит скрытую реализацию пропущенной функции в другой фазе.

QED.

### Уточнение границ применимости

Теорема задает минимальность в смысле функциональной декомпозиции универсальной
адаптации. Она не запрещает доменные компрессии интерфейса (например, 4-фазные
представления), если функциональные классы остаются реализованы неявно.

---

## 5. Связь с сокращенными моделями

Сокращенные схемы (OODA, PDCA и др.) трактуются как компрессия CF-6:

- `observe` = `collect` + часть `analyze`;
- `orient/plan` = `analyze` + `forecast`;
- `decide` = `decide`;
- `act` = `implement` + часть `evaluate`.

Они не опровергают CF-6, а используют агрегированное представление.

---

## 6. Связь CF-6 и LC-6

Если `CF-6` задает микроструктуру одного шага редукции, то `LC-6` задает
макро-геометрию накопления шагов во времени:

`x_(k+1) = CF6(x_k)`.

Последовательность `{CF6_k}` формирует наблюдаемую фазовую динамику `LC-6`.

Функциональное разграничение:

- `CF-6`: как выполняется один шаг изменения;
- `LC-6`: в каком режиме находится система как итог множества шагов.

Канонический мост изменения `LC` задается только через `CF6` (Evaluate):
фазы `CF1..CF5` не изменяют `LC` напрямую.

---

## 7. Отображение Psi: CF -> LC

Вводится интерпретационное отображение:

`Psi: stats({CF6_k}) -> LC_phase`.

`Psi` использует статистику выполнения циклов (ошибки, стоимость, устойчивость,
долю успешных реализаций и т.д.) для классификации текущей `LC`-фазы.

Важно:
`Psi` — профиль интерпретации (policy/model dependent), а не универсальная
жесткая биекция.
Следовательно, утверждения вида `LC-6 ≅ (CF-6)^n` допустимы только как
интуитивная метафора масштаба, но не как канонический изоморфизм.

---

## 8. Канонический итог

В рамках CDM v2:

- `CF-6` — минимальный оператор адаптивной необратимой редукции;
- `LC-6` — макро-фазовая геометрия повторяющегося `CF-6`;
- обе схемы интерпретируются как два масштаба одной динамической структуры.
