# Спецификация: CDM v2 — Минимальность CF-6 (canonical short form)

> Краткая каноническая форма для цитирования в спецификациях CDM.
> Полная версия: `CDM-v2-Minimality-Theorem`.

---

## 1. Аксоматический минимум

Пусть адаптивная система:

`S = (Omega, R, M, Pi)`, при ограничении `R < |Omega|`.

Следствия:

- полное разрешение альтернатив за один акт невозможно;
- разрешение неопределенности выполняется итеративными циклами;
- редукция альтернатив в цикле необратима при ненулевой стоимости возврата.

Целевая функция:

`min_(Pi) E[tau_future]`.

---

## 2. Каноническая форма CF-6

Минимальный функциональный оператор адаптивной редукции:

`CF-6 = (collect, analyze, forecast, decide, implement, evaluate)`.

Операторная запись редукции шага:

`Phi_k: T_k -> T_(k+1)`, `|T_(k+1)| < |T_k|`.

Опциональная профильная метрика редукции:

`tau_red(k) = log(|T_k| / |T_(k+1)|)`.

`tau_red` не заменяет канонический `tau` в спецификациях CF/LC.

Операторы:

- `C: W -> D`;
- `A: D -> S_hat`;
- `F: S_hat -> {T_i}`;
- `Dc: {T_i} -> T*`;
- `I: T* -> W'`;
- `E: (W', M) -> DeltaM`.

---

## 3. Тезис минимальности

Для универсальной адаптивной системы цикл с числом функциональных классов `< 6`
неполон: при удалении любого элемента из `{C, A, F, Dc, I, E}` теряется
обязательная функция адаптации.

Доменные 4-фазные представления допускаются как компрессии интерфейса, но не
как устранение функций.

---

## 4. Связь с LC-6

`CF-6` — микроструктура одного шага изменения.  
`LC-6` — макро-фазовая геометрия повторяющихся шагов:

`x_(k+1) = CF6(x_k)`.

Интерпретационное отображение:

`Psi: stats({CF6_k}) -> LC_phase`.

`Psi` является policy-dependent профилем классификации фаз.
Каноническое изменение `LC` выполняется только через `CF6` (Evaluate bridge).
Формула `LC-6 ≅ (CF-6)^n` не используется как строгий изоморфизм.

---

## 5. Канонический итог

- `CF-6` — минимальный оператор адаптивной необратимой редукции.
- `LC-6` — макро-динамика системы как результат повторения `CF-6`.
- `LC-6` и `CF-6` — два масштаба одной структуры динамики.
