# Примеры: Doubt

## 1. Базовый расчет

Пусть:

- `A=4.0`, `A0=3.0`
- `T=1.1`, `T0=0.8`
- `alpha=3.0`
- `C_err=1.4`

Тогда:

- `A/(A+A0)=4/7 ~ 0.571`
- `sigma(3*(0.8-1.1))=sigma(-0.9) ~ 0.289`
- `Doubt ~ 1.4 * 0.571 * 0.289 ~ 0.231`

---

## 2. Интерпретация в CF

1. `Doubt` низок -> стандартный проход `forecast -> decide`.
2. `Doubt` средний/высокий -> расширенный `forecast` и доп. валидация альтернатив.
3. `Doubt > theta_domain` -> запуск `MetaChangeFlow` (если задано профилем).

---

## 3. Анти-примеры

1. Расчет без явного набора альтернатив -> невалидно.
2. Сравнение разных контекстов в одном `A` без нормализации -> невалидно.
3. Переход в `implement` только по `Doubt`, без гейтов -> неканонично.

---

## 4. v2 advanced profile illustration

При использовании `Doubt_v2` (немонотонный профиль по `T`):

1. Низкий `T` -> низкий сигнал сомнения (возможна ложная уверенность).
2. Средний `T` -> пик сомнения (напряжение выбора при росте вариантов).
3. Высокий `T` -> снижение сомнения до управляемого уровня.

Это удобно для креативных и исследовательских контуров.

С учетом опыта (`E_conf`):
- два контура с одинаковыми `A` и `T` могут иметь разный `Doubt_v2`;
- контур с более высоким `E_conf` (устойчивые релевантные `+1`) получает ниже итоговый сигнал сомнения.

---

## 5. Human-readable note (non-normative)

Для интуитивного объяснения (не канон):

1. Низкий сигнал сомнения при низкой реальной емкости может выглядеть как "Dunning-Kruger-like" режим.
2. Рост сигнала сомнения в переходной зоне может напоминать "impostor-like" напряжение выбора.
3. Зрелая операторная емкость соответствует режиму управляемой уверенности.

Эти ярлыки используются только как объяснительная метафора для операторов и не являются диагностическими категориями модели.
