# Модель: Subjectivity (core)

## 1. Формула

Базовая форма:

`S = f(sum_i(W_i * R_i(C) * D(T_A_i(C), T_B_i(C)) / (Q(T_A_i(C)) * Q(T_B_i(C)))))`

где:

- `W_i > 0` — вес i-го инструмента;
- `R_i(C) in [0,1]` — релевантность в контексте `C`;
- `D(...) in [0,1]` — нормированное различие;
- `Q(...) in (0,1]` — качество/эффективность инструмента;
- `f(.)` — функция сжатия/нормировки.

---

## 2. Контекст

`C` включает минимум:

1. область применения;
2. цель/тип задачи;
3. ограничения среды.

`R_i(C)=1` только при полной релевантности инструмента в данном контексте.

---

## 3. Рекомендуемые функции `f(.)`

- линейная: `f(x)=x`;
- логарифмическая: `f(x)=ln(1+x)` (базовый выбор);
- сигмоидная: `f(x)=1-exp(-k*x)`;
- квадратичная saturating: `f(x)=1-1/(1+x^2)`.

---

## 4. Защита от аномалий

1. Использовать `Q_min > 0` (обычно `0.01..0.05`).
2. При `R_i(C)=0` элемент исключается из суммы.
3. Нормировать веса (`sum W_i = 1` или `100`) для межзадачного сравнения.

---

## 5. Интерпретация

- `S` — непрерывная величина;
- рост `S` означает рост расхождения траекторий;
- декомпозиция по i показывает вклад источников субъективности.

---

## 6. Статус

Документ является общей междоменной моделью.
Доменные пороги и политики применения задаются в профильных документах.

---

## 7. Операционная заметка для ИС (Q и D)

Для информационных систем рекомендуется фиксировать единый operational профиль:

1. `Q(T)` (качество инструмента) считать как агрегат нормированных компонент:
- `q_success` (доля успешных релевантных прогонов),
- `q_latency` (нормированная задержка),
- `q_cost` (нормированная стоимость),
- `q_reliability` (устойчивость/ошибки).

Пример:
`Q = clamp(w1*q_success + w2*q_latency + w3*q_cost + w4*q_reliability, Q_min, 1)`,
где `sum(wk)=1`.

2. `D(T_A, T_B)` (различие траекторий) считать по разнице outcome-профилей:
- различие доступных операторов,
- различие ожидаемых классов результата,
- различие устойчивости/стоимости траектории.

Пример нормировки:
`D = clamp(beta1*d_ops + beta2*d_result + beta3*d_runtime, 0, 1)`,
где `sum(beta_k)=1`.

3. Для сопоставимости между командами:
- фиксировать веса `wk`, `beta_k` в domain profile;
- фиксировать одно и то же окно наблюдений;
- хранить версию профиля расчета вместе с результатом `S`.
