# Profile: Subjectivity and Creativity in Software

## 1. Назначение

Профиль фиксирует применение связки `R + S(A,Ref,C)` для оценки
нестандартности и креативности инженерного решения в разработке ПО.

## 2. Контекст кейса (sorting)

- Область: разработка ПО.
- Цель: сортировка списка из 1 млн целых чисел.
- Ограничения: Python, одноядерный CPU, память <= 64 MB.
- Критерий результата `R`: время выполнения (меньше — лучше).

## 3. Эталонный агент

Эталон задается по
[[fcdm-core/theory/cdm/Specifications/System/Subjectivity-Reference-Agent-Profile|Subjectivity-Reference-Agent-Profile]].

Референсный baseline: стандартный high-quality набор операторов сортировки,
релевантный ограничениям контекста.

## 4. Сравнение решений

### A (нестандартный)

Гибрид: radix (старшие разряды) + insertion для малых блоков.

- наблюдаемый результат: `0.61s`;
- относительно baseline: `R(A) > R(Ref)`;
- измеренная субъективность: `S(A,Ref)` высокая (`~1.91`, иллюстративный расчет).

### B (шаблонный)

`sorted(data)` в baseline-стиле.

- наблюдаемый результат: около baseline (`~0.80..0.81s`);
- `S(B,Ref)` близко к нулю (`~0.04`, иллюстративный расчет).

## 5. Вывод

При сопоставимых условиях:

- `R(A) >= R(Ref)` и высокий `S(A,Ref)` -> нестандартное и эффективное решение;
- `R(B) ~= R(Ref)` и низкий `S(B,Ref)` -> шаблонное решение.

Оценка креативности выполняется только по связке `R + S`,
а не по `S` отдельно.

## 6. Ограничение

Численные значения в кейсе являются демонстрационными.
Для production-оценки обязательны единые правила расчета `R`, `S` и единый `Ref`.
