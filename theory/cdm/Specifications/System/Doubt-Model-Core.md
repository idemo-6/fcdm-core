# Модель: Doubt (core)

## 1. Формула (v1 simplified baseline)

Базовая форма:

`Doubt = C_err * (A / (A + A0)) * sigma(alpha * (T0 - T))`

где:

- `A = sum_j(v_j)` — взвешенная сумма альтернатив;
- `T = sum_i(W_i * Q_i)` — агрегированная операторная емкость;
- `C_err >= 1` — цена ошибки;
- `A0 > 0` — полунасыщение по альтернативам;
- `alpha > 0` — крутизна перехода;
- `T0` — порог операторной емкости;
- `sigma(x)=1/(1+exp(-x))`.

Смысл выбранного знака:
- при росте `T` (операторной емкости) вклад сигнала сомнения снижается;
- при `T << T0` сомнение усиливается.

Эта форма является упрощенным baseline-профилем:
- удобна для стабильных доменов с предсказуемой операторной динамикой;
- используется, когда достаточно монотонного влияния `T`.

## 1.1 Формула (v2 recommended advanced profile with Experience)

Для доменов, где сомнение сначала растет с расширением компетентности,
а затем снижается при зрелой операторной емкости, рекомендуется немонотонный профиль:

`Doubt_v2 = C_err * (A / (A + A0)) * h(T) * m(E_conf)`

где:

`h(T) = sigma(k1 * (T - T1)) * (1 - sigma(k2 * (T - T2)))`, `T1 < T2`, `k1,k2 > 0`.

`E_conf in [0,1]` — индекс подтвержденной освоенности операторного набора по локальному опыту (`Experience`, преимущественно `+1` записи в релевантном `C_active` и `LC_phase`).
Рекомендуемая междоменная формализация индекса: [[fcdm-core/theory/cdm/Specifications/Experience/Experience-Model-Core|Experience-Model-Core]] (раздел `E_conf`).

`m(E_conf) in (0,1]` — модуль доверия к операторной емкости.
Рекомендуемый baseline:
`m(E_conf) = 1 - gamma * E_conf`, где `gamma in [0,1)`.

Интерпретация:
- низкий `T` -> низкий активный сигнал сомнения (зона ложной уверенности);
- средний `T` -> рост сигнала сомнения (переходная зона выбора);
- высокий `T` -> снижение сигнала сомнения (зона управляемого выбора).
- при прочих равных рост `E_conf` снижает `Doubt_v2` (операторы "подтверждены опытом", а не только декларированы).

Профиль `v2` рекомендуется для:
- креативных/исследовательских задач;
- ситуаций с выраженной фазой "расширения вариантов";
- образовательных и экспертных контуров, где наблюдается переходная зона сомнения.

---

## 2. Параметры

Ориентиры (профильно настраиваются):

- `A0`: `2..5`
- `alpha`: `1..5`
- `T0`: `0.5..1.5`
- `C_err`: `1..3`

Для `v2` дополнительно:
- `T1`: начало зоны роста сомнения;
- `T2`: начало зоны снижения сомнения (`T2 > T1`);
- `k1`, `k2`: крутизна фронтов.
- `gamma`: сила влияния опыта на снижение сомнения;
- `E_conf`: индекс, вычисляемый из `Experience` в текущих `C_active`, `LC_phase`.

---

## 3. Интерпретация

- `Doubt ~ 0`: низкий сигнал сомнения (возможна ложная уверенность);
- `Doubt ~ 1` (или высокий профильный диапазон): высокий сигнал сомнения, требующий пересмотра.

Интерпретация диапазонов фиксируется доменным профилем.

Монотонность по `T`:
- при фиксированных `A, C_err` и `alpha > 0` рост `T` не увеличивает `Doubt`.

Для `v2`:
- зависимость от `T` немонотонна (inverted-U профиль).
- зависимость от `E_conf` монотонно убывающая: рост подтвержденного опыта снижает сомнение.

---

## 4. Динамическая настройка

Параметры могут адаптироваться по графу опыта:

- преобладание `+1` ассоциаций -> снижение `A0`, `T0`;
- рост `-1` ассоциаций -> повышение `C_err`;
- высокая контекстная динамика -> увеличение `alpha`.

Для `v2`:
- рост доли устойчивых `+1` в релевантных записях опыта -> рост `E_conf`;
- рост деградирующих `-1` и нестабильных `0` в релевантном контуре -> снижение `E_conf`;
- изменение `LC_phase` требует пересчета `E_conf` для новой фазовой сигнатуры.

---

## 5. Условия корректности

1. Альтернативы должны быть определены в одном контексте `C`.
2. `A`, `T` считаются по явно заданным правилам агрегации.
3. Для `v2` параметры `T1,T2,k1,k2,gamma` должны фиксироваться в domain profile.
4. `E_conf` должен вычисляться только по релевантному `Experience` (`C_active`, `LC_phase`, валидные `ExpKey`).
5. Решения по `Doubt` не допускаются без evidence и phase-gate.
