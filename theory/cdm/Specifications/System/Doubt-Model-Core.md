# Модель: Doubt (core)

## 1. Формула

Базовая форма:

`Doubt = C_err * (A / (A + A0)) * sigma(alpha * (T - T0))`

где:

- `A = sum_j(v_j)` — взвешенная сумма альтернатив;
- `T = sum_i(W_i * Q_i)` — агрегированная операторная емкость;
- `C_err >= 1` — цена ошибки;
- `A0 > 0` — полунасыщение по альтернативам;
- `alpha > 0` — крутизна перехода;
- `T0` — порог операторной емкости;
- `sigma(x)=1/(1+exp(-x))`.

---

## 2. Параметры

Ориентиры (профильно настраиваются):

- `A0`: `2..5`
- `alpha`: `1..5`
- `T0`: `0.5..1.5`
- `C_err`: `1..3`

---

## 3. Интерпретация

- `Doubt ~ 0`: низкий сигнал сомнения (возможна ложная уверенность);
- `Doubt ~ 1` (или высокий профильный диапазон): высокий сигнал сомнения, требующий пересмотра.

Интерпретация диапазонов фиксируется доменным профилем.

---

## 4. Динамическая настройка

Параметры могут адаптироваться по графу опыта:

- преобладание `+1` ассоциаций -> снижение `A0`, `T0`;
- рост `-1` ассоциаций -> повышение `C_err`;
- высокая контекстная динамика -> увеличение `alpha`.

---

## 5. Условия корректности

1. Альтернативы должны быть определены в одном контексте `C`.
2. `A`, `T` считаются по явно заданным правилам агрегации.
3. Решения по `Doubt` не допускаются без evidence и phase-gate.
